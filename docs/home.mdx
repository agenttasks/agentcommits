title: "Overview"

Agent Commits defines a lightweight file format (`COMMIT.md`) that tells AI agents how a project wants commits formatted. It extends [Conventional Commits 1.0.0](https://www.conventionalcommits.org/en/v1.0.0/) with agent-specific concerns — attribution, commit splitting, and per-project configuration.

## Why Agent Commits?

AI agents are writing more code and making more commits, but every agent handles commit messages differently. Projects have no standard way to tell agents their commit conventions. Agent Commits solves this with a single file that any agent can read.

**For project maintainers**: Declare your commit conventions once and have every agent follow them — no tool-specific configuration needed.

**For agent developers**: Support for Agent Commits lets your agent produce commits that match project expectations out of the box.

**For teams and enterprises**: Enforce consistent commit attribution, structured changelogs, and automated versioning across agent-generated code.

## What does Agent Commits enable?

- **Consistent formatting**: Agents follow project-specific commit conventions (types, scopes, description style) without per-agent configuration.
- **Agent attribution**: Standardized footers identify which agent, model, and human were involved in each commit.
- **Automated tooling**: Conventional commit format enables release-please, semantic versioning, and changelog generation.
- **Interoperability**: The same `COMMIT.md` works across Claude Code, Cursor, GitHub Copilot, and any agent that supports the spec.

## Open development

The Agent Commits specification was developed by [Anthropic](https://www.anthropic.com/) and released as an open standard. The format builds on the widely-adopted [Conventional Commits](https://www.conventionalcommits.org/) specification and is inspired by the [Agent Skills](https://agentskills.io) format.
